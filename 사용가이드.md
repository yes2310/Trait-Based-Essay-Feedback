# LEAF 작업 폴더 사용 가이드

## 1. 목적

이 폴더는 LEAF 데이터셋만 따로 떼어 다른 위치로 옮겨도 실행할 수 있도록 만든 독립 작업 폴더입니다.

즉, 상위 PromptAES2 레포 경로를 기억할 필요 없이 이 폴더만 있으면 됩니다.

## 2. 내부 구조

- `data/leaf_merged.csv`
  - 현재 학습에 사용하는 정규화된 LEAF 데이터
- `data/leaf_merged_3class.csv`
  - trait 점수를 `1/2/3`으로 재매핑한 3-class 실험용 LEAF 데이터
- `legacy_data/filtered_leaf3000.csv`
  - 예전에 루트 레포에 있던 보조 LEAF 데이터
- `runtime/promptaes2/`
  - 실행에 사용하는 로컬 PromptAES2 코드
- `results/`
  - 학습 결과 저장 경로
- `run_pretrain.sh`
  - trait 사전학습 실행 스크립트
- `run_holistic.sh`
  - holistic 학습 실행 스크립트
- `run_trait_score.sh`
  - trait-score 학습 실행 스크립트
- `_common.sh`
  - 공통 실행 함수
- `setup_uv.sh`
  - `uv` 가상환경 생성 및 의존성 설치 스크립트
- `install_requirements.sh`
  - 기존 이름 호환용 래퍼 스크립트
- `run_all.sh`
  - 전체 파이프라인 일괄 실행 스크립트

## 3. 어떻게 동작하는가

실행 스크립트는 공통적으로 아래 방식으로 동작합니다.

1. 스크립트 자신의 위치를 기준으로 작업 폴더 경로를 계산합니다.
2. `uv`로 만든 로컬 가상환경 `.venv/`를 사용합니다.
3. `runtime/`를 `PYTHONPATH`에 추가합니다.
4. 그래서 시스템 전역의 PromptAES2가 아니라 이 폴더 안의 `runtime/promptaes2/` 코드를 사용합니다.
5. 데이터 경로와 결과 경로도 모두 이 폴더 기준 절대 경로로 전달합니다.

따라서 `leaf_workspace/` 전체를 다른 위치로 옮겨도 스크립트 경로만 유지되면 그대로 실행됩니다.

## 4. 실행 순서

### 4-1. `uv` 설치

`uv`가 없다면 먼저 설치합니다.

```bash
python3 -m pip install uv
```

### 4-2. 가상환경 및 패키지 설치

```bash
./setup_uv.sh
```

기존 이름을 유지하고 싶으면 아래도 가능합니다.

```bash
./install_requirements.sh
```

### 4-3. trait 사전학습

```bash
./run_pretrain.sh
```

기본 스크립트는 `--predefined_split_column split`을 포함하므로 CSV의 `split` 컬럼에 있는 `train/dev/test` 공식 분할을 그대로 사용합니다.
또한 현재 스크립트는 `data/leaf_merged_3class.csv`를 사용하므로 trait 점수는 `1,2 -> 1`, `3 -> 2`, `4,5 -> 3`으로 변환된 상태입니다.

결과 저장 위치:

- `results/trait_pretrain_3class/`

### 4-4. holistic 학습

```bash
./run_holistic.sh
```

이 단계도 동일하게 `--predefined_split_column split`을 사용하므로 pretrain과 같은 공식 분할 규칙을 유지합니다.

결과 저장 위치:

- `results/holistic_3class/`

### 4-5. trait-score 학습

```bash
./run_trait_score.sh
```

이 단계 역시 `split` 컬럼의 공식 `train/dev/test` 분할을 그대로 사용합니다.

결과 저장 위치:

- `results/trait_score_3class/`

### 4-6. 한 번에 전체 실행

```bash
./run_all.sh
```

이 스크립트는 아래 순서로 자동 실행합니다.

1. `setup_uv.sh`
2. `run_pretrain.sh`
3. `run_holistic.sh`
4. `run_trait_score.sh`

## 5. trait 그룹 구성

현재 권장 그룹은 아래와 같습니다.

```text
alignment_with_topic,arguments_supporting_details:2;clarity_of_view_point,spelling_grammar_style:2
```

이렇게 나눈 이유는:

- `alignment_with_topic` 와 `arguments_supporting_details` 는 내용/근거 축에 가깝고
- `clarity_of_view_point` 와 `spelling_grammar_style` 는 표현/명확성 축에 가깝기 때문입니다.

또한 그룹이 2개 이상이어야 hetero 경로가 의미 있게 작동합니다.

## 6. 현재 주의점

### 6-1. 공식 split 사용 방식

현재 런타임은 `--predefined_split_column split`이 주어지면 CSV의 `split` 컬럼을 공식 train/dev/test 분할로 그대로 사용합니다.
`split_by_column`은 이 용도가 아니라 별도 group-by 실행용 옵션입니다.

### 6-2. 점수 불균형

`total_score` 가 `12~14` 구간에 많이 몰려 있어서,
드문 점수 구간은 holistic 분류가 불안정할 수 있습니다.

## 7. 압축 및 전달

이 폴더는 그대로 압축해서 다른 위치로 옮기면 됩니다.

압축본 예시:

- `leaf_workspace_portable_YYYYMMDD.zip`

압축을 푼 뒤에는 이 폴더 안에서 바로 아래 순서로 실행하면 됩니다.

```bash
python3 -m pip install uv
./setup_uv.sh
./run_pretrain.sh
./run_holistic.sh
./run_trait_score.sh
```

또는 한 번에:

```bash
python3 -m pip install uv
./run_all.sh
```
