# LEAF 작업 폴더 사용 가이드

## 1. 목적

이 폴더는 LEAF 데이터셋만 따로 떼어 다른 위치로 옮겨도 실행할 수 있도록 만든 독립 작업 폴더입니다.

즉, 상위 PromptAES2 레포 경로를 기억할 필요 없이 이 폴더만 있으면 됩니다.

## 2. 내부 구조

- `data/raw/leaf--main/`
  - 새로 받은 LEAF 원본 split CSV와 rubric 파일
- `data/raw/leaf--main.zip`
  - 원본 ZIP 사본
- `data/leaf_merged.feedback4_only.csv`
  - 기존 4개 feedback dimension 기준 호환 CSV
- `data/leaf_merged.csv`
  - 기존 점수 컬럼과 새 10개 trait를 합친 통합 CSV
- `scripts/prepare_leaf_dataset.py`
  - ZIP에서 raw/merged 데이터를 재생성하는 준비 스크립트
- `legacy_data/filtered_leaf3000.csv`
  - 예전에 루트 레포에 있던 보조 LEAF 데이터
- `runtime/promptaes2/`
  - 실행에 사용하는 로컬 PromptAES2 코드
- `results/`
  - 학습 결과 저장 경로
- `run_pretrain.sh`
  - trait 사전학습 실행 스크립트
- `run_holistic.sh`
  - holistic 학습 실행 스크립트
- `run_trait_score.sh`
  - trait-score 학습 실행 스크립트
- `_common.sh`
  - 공통 실행 함수
- `setup_uv.sh`
  - `uv` 가상환경 생성 및 의존성 설치 스크립트
- `install_requirements.sh`
  - 기존 이름 호환용 래퍼 스크립트
- `run_all.sh`
  - 전체 파이프라인 일괄 실행 스크립트

## 3. 어떻게 동작하는가

실행 스크립트는 공통적으로 아래 방식으로 동작합니다.

1. 스크립트 자신의 위치를 기준으로 작업 폴더 경로를 계산합니다.
2. `uv`로 만든 로컬 가상환경 `.venv/`를 사용합니다.
3. `runtime/`를 `PYTHONPATH`에 추가합니다.
4. 그래서 시스템 전역의 PromptAES2가 아니라 이 폴더 안의 `runtime/promptaes2/` 코드를 사용합니다.
5. 데이터 경로와 결과 경로도 모두 이 폴더 기준 절대 경로로 전달합니다.

따라서 `leaf_workspace/` 전체를 다른 위치로 옮겨도 스크립트 경로만 유지되면 그대로 실행됩니다.

## 3-1. 새 데이터셋 반영 방식

새로 받은 `leaf--main.zip`에는 두 종류의 라벨이 들어 있습니다.

1. root split
   - `trait_1` ~ `trait_10`
2. `LEAF-Feedback-dimension/`
   - `alignment_with_topic`
   - `spelling_grammar_style`
   - `clarity_of_view_point`
   - `arguments_supporting_details`
   - `overall`

현재 작업 폴더는 두 소스를 `(ID, split)` 기준으로 합쳐서 사용합니다.

- `total_score`
  - 기존 파이프라인 호환을 위해 4개 feedback dimension의 합으로 유지
- `feedback_dimension_overall`
  - raw `overall` 값을 별도 보존
- 새 semantic trait 10개
  - `trait_1`~`trait_10`을 읽기 쉬운 이름으로 추가

새 ZIP에서 다시 데이터를 만들고 싶다면 아래 명령을 사용합니다.

```bash
python3 scripts/prepare_leaf_dataset.py --zip-path /path/to/leaf--main.zip
```

이 명령은 아래를 수행합니다.

1. ZIP을 `data/raw/` 아래로 복사
2. 원본 split CSV를 `data/raw/leaf--main/`에 풀기
3. `data/leaf_merged.feedback4_only.csv` 생성
4. `data/leaf_merged.csv` 생성
5. 형식이 깨진 원본 rubric을 `rubrics.normalized.json`으로 정리

## 4. 실행 순서

### 4-1. `uv` 설치

`uv`가 없다면 먼저 설치합니다.

```bash
python3 -m pip install uv
```

### 4-2. 가상환경 및 패키지 설치

```bash
./setup_uv.sh
```

기존 이름을 유지하고 싶으면 아래도 가능합니다.

```bash
./install_requirements.sh
```

### 4-3. trait 사전학습

```bash
./run_pretrain.sh
```

기본 스크립트는 10개의 semantic trait를 모두 사전학습 대상으로 사용합니다.
또한 `--predefined_split_column split`을 포함하므로 CSV의 `split` 컬럼에 있는 `train/dev/test` 공식 분할을 그대로 사용합니다.

결과 저장 위치:

- `results/trait_pretrain/`

### 4-4. holistic 학습

```bash
./run_holistic.sh
```

이 단계는 10개 trait embedding을 `표현(3) / 조직(4) / 내용(3)` 3개 그룹으로 묶어 holistic 점수를 예측합니다.
기본 실행 스크립트는 `canonical_moe + evolution_stage=full + warmup_epochs=3 + class_balance_mode=loss_and_sampler`를 사용합니다.
동일하게 `--predefined_split_column split`을 사용하므로 pretrain과 같은 공식 분할 규칙을 유지합니다.

결과 저장 위치:

- `results/holistic/`

### 4-5. trait-score 학습

```bash
./run_trait_score.sh
```

이 단계는 10개 semantic trait 각각을 target으로 학습합니다.
역시 `split` 컬럼의 공식 `train/dev/test` 분할을 그대로 사용합니다.

결과 저장 위치:

- `results/trait_score/`

### 4-6. 한 번에 전체 실행

```bash
./run_all.sh
```

이 스크립트는 아래 순서로 자동 실행합니다.

1. `setup_uv.sh`
2. `run_pretrain.sh`
3. `run_holistic.sh`
4. `run_trait_score.sh`

## 5. trait 그룹 구성

현재 권장 그룹은 아래와 같습니다.

```text
grammar_accuracy,appropriateness_of_word_use,elasticity_of_sentence_expression:3;appropriateness_of_structure_within_a_paragraph,adequacy_of_inter_paragraph_structure,consistency_of_structure,appropriateness_of_portion_size:4;clarity_of_topic,specificity_of_explanation,creativity_of_thought:3
```

이렇게 나눈 이유는:

- 표현 그룹:
  `grammar_accuracy`, `appropriateness_of_word_use`, `elasticity_of_sentence_expression`
- 조직 그룹:
  `appropriateness_of_structure_within_a_paragraph`, `adequacy_of_inter_paragraph_structure`, `consistency_of_structure`, `appropriateness_of_portion_size`
- 내용 그룹:
  `clarity_of_topic`, `specificity_of_explanation`, `creativity_of_thought`

또한 현재 hetero 단계는 3개 group token을 직접 self-attention으로 contextualize하므로, 이 3축 분해가 가장 자연스럽습니다.

## 6. 현재 주의점

### 6-1. 공식 split 사용 방식

현재 런타임은 `--predefined_split_column split`이 주어지면 CSV의 `split` 컬럼을 공식 train/dev/test 분할로 그대로 사용합니다.
`split_by_column`은 이 용도가 아니라 별도 group-by 실행용 옵션입니다.

### 6-2. 점수 불균형

`total_score` 가 `12~14` 구간에 많이 몰려 있어서,
드문 점수 구간은 holistic 분류가 불안정할 수 있습니다.
그래서 현재 실행 스크립트는 train split 기준 class weight와 weighted sampler를 함께 켭니다.

### 6-3. trait_10 이상치

root dataset의 raw `trait_10`에는 `0` 값이 4개 있습니다.
원본 rubric은 1~5만 정의하고 있으므로, 정규화 컬럼 `creativity_of_thought`에서는 이 값을 결측치로 처리합니다.
따라서 학습에는 raw `trait_10`보다 `creativity_of_thought`를 사용하는 편이 안전합니다.

## 7. 압축 및 전달

이 폴더는 그대로 압축해서 다른 위치로 옮기면 됩니다.

압축본 예시:

- `leaf_workspace_portable_YYYYMMDD.zip`

압축을 푼 뒤에는 이 폴더 안에서 바로 아래 순서로 실행하면 됩니다.

```bash
python3 -m pip install uv
./setup_uv.sh
./run_pretrain.sh
./run_holistic.sh
./run_trait_score.sh
```

또는 한 번에:

```bash
python3 -m pip install uv
./run_all.sh
```
